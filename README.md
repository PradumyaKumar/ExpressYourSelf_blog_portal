# Blog Express — Monorepo

A monorepo for a blog platform: a Next.js frontend plus Node/TypeScript microservices for `author`, `blog`, `social`, and `user`.

## Repository layout

- `Frontend/` — Next.js frontend (TypeScript)
- `services/author` — Author service
- `services/blog` — Blog service
- `services/social` — Social service
- `services/user` — User service

## Prerequisites

- Node.js (v16+ recommended)
- npm or yarn (or pnpm)
- MongoDB / Postgres or other DB as required by services
- RabbitMQ (if using message broker in this repo)

## Quick setup

1. Install dependencies for the frontend:

```bash
cd Frontend
npm install
# or `yarn` / `pnpm install`
```

2. Install dependencies for each service (example for `blog`):

```bash
cd services/blog
npm install

# Repeat for services/author, services/social, services/user
```

## Environment variables

Each service and the frontend use environment variables. Create a `.env` file in the corresponding folder with at least the following (example):

Frontend/.env

```
# Example
NEXT_PUBLIC_API_URL=http://localhost:5000
```

services/blog/.env (example)

```
PORT=5000
DATABASE_URL=mongodb://localhost:27017/blog-db
RABBITMQ_URL=amqp://localhost
JWT_SECRET=your_jwt_secret
```

Adjust names/values to match each service's needs (`author`, `social`, `user`). Check the service `src` folders for exact env variable names.

## Running locally

- Start frontend:

```bash
cd Frontend
npm run dev
```

- Start a service (example `blog`):

```bash
cd services/blog
npm run dev
```

Run each service in its own terminal. Ports and commands might vary by service — check each `package.json` under `services/*`.
